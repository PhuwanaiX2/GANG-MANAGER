Mobile Behavior

มือถือไม่ใช่เวอร์ชันย่อของ desktop แต่เป็น task list ที่กดง่าย

Member ใช้งาน mobile มาก จึงให้ action สำคัญอยู่บน/ล่างที่นิ้วถึง และ table ทุกตัวต้องมี card equivalent

Navigation

mobile ใช้ drawer ที่จัด group เดียวกับ desktop และมี badge คิวค้าง ไม่ใช้ bottom nav ที่ยัดทุกอย่างจนอ่านไม่ออก

Tables

members, finance, attendance roster แปลงเป็น cards โดยเก็บ primary action เช่น เปิด profile, approve, mark status ไว้บน card

Forms

Density

บนมือถือซ่อน chart generic ก่อน แต่เก็บ ranked insight/attention queue เพราะเป็นสิ่งที่ตอบว่าต้องทำอะไรต่อ
